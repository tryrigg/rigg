## shell

Runs a shell command.

```yaml
- id: test
  type: shell
  with:
    command: npm test
    result: text # none | text | json (default: text)
```

`result` controls stdout capture: `none` discards it, `text` captures as string, `json` parses as structured data.

## codex

Two actions are supported: `run` and `review`.

### run

Generates or modifies code.

```yaml
- id: implement
  type: codex
  with:
    action: run
    prompt: Add input validation.
    model: gpt-5.4
```

`steps.<id>.result` is always the final plain-text response.

### review

Scans diffs and returns review findings.

```yaml
- id: review
  type: codex
  with:
    action: review
    model: gpt-5.4
    review:
      target:
        type: uncommitted # uncommitted | base | commit
```

Target variants:

- `type: uncommitted`
- `type: base` with `branch`
- `type: commit` with `sha`

Review always returns the built-in structured result below:

```yaml
result:
  findings:
    - title: string
      body: string
      confidence_score: number
      priority: integer
      code_location:
        absolute_file_path: string
        line_range:
          start: integer
          end: integer
  overall_correctness: string
  overall_explanation: string
  overall_confidence_score: number
```

## write_file

Writes content to a file.

```yaml
- id: save
  type: write_file
  with:
    path: ${{ inputs.output_path }}
    content: ${{ steps.draft.result }}
```

## Result access

- `codex run`: `steps.draft.result`
- `codex review`: `steps.review.result.findings`
