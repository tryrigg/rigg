## rigg init

Create a `.rigg/` directory with example workflow files.

```bash
rigg init
```

## rigg validate

Validate all `.rigg/*.yaml` workflow files. Reports syntax errors, unknown fields, invalid references, and schema violations.

```bash
rigg validate
```

## rigg run

Execute a workflow by ID.

```bash
rigg run <workflow_id> [--input key=value] [--auto-continue]
```

| Flag                | Description                                                                                                    |
| ------------------- | -------------------------------------------------------------------------------------------------------------- |
| `--input key=value` | Set input parameter (repeatable). Values are JSON-parsed when possible.                                        |
| `--auto-continue`   | TTY-only mode that skips step barriers, while still pausing for approvals, questions, and other input prompts. |

On a TTY, `rigg run` opens a stateful terminal UI with:

- a run header
- a live running view for the active step output and codex activity
- a waiting view for barriers and interactions
- a final view that shows the last completed step result

With `--auto-continue`, the same TTY UI stays active, but step barriers advance automatically and the UI emits a muted `auto-continue: Next: ...` line for each released barrier. Real runtime interactions such as approvals and user input still pause normally.

Use `Ctrl-C` to interrupt the active step. Non-TTY execution is rejected because barriers and workflow input prompts are interactive, including when `--auto-continue` is set.

Workflows are discovered from the nearest `.rigg/` directory.
